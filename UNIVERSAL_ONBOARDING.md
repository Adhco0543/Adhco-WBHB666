# Universal Adaptive Onboarding System

## What Changed

Your app is now **truly universal** for ANY business. It's no longer hardcoded for carpenters or salons. Instead, it:

1. **Asks users what they do** (in their own words)
2. **Uses AI to understand their situation** (Claude analyzes their description)
3. **Generates personalized tasks** (customized task buttons for their work)
4. **Adapts the entire experience** (dashboard, chat, recommendations)

---

## How It Works

### Step 1: Adaptive Onboarding
User enters: *"I'm a stay-at-home mom who tutors kids in math online"*

Claude analyzes and returns:
- **Work Type:** service
- **Revenue Model:** hourly
- **Team Size:** solo
- **Work Location:** home
- **Suggested Tasks:** ["Schedule Lesson", "Track Hours", "Create Invoice", "Manage Student Progress"]
- **Pain Points:** [scheduling, client_communication, invoicing]
- **Professional Context:** "You provide educational services to students remotely on flexible hours."
- **Adaptive Prompt:** "I should help you manage student schedules, track billable hours, communicate with parents, and handle invoicing efficiently."

### Step 2: Personalized Dashboard
Instead of hardcoded buttons ("Make Quote", "Materials List", etc.), the dashboard shows:
- **Your Profile Section:** Work Type, Revenue Model, Team Size, Work Location
- **Your Personalized Tasks:** 4 custom buttons based on the AI analysis
- **Focus Areas:** Pre-populated pain points
- **Assistant Memory:** Adaptive prompt to guide the assistant

### Step 3: Context-Aware Chat
When the user chats, the assistant knows:
- What they do
- How they make money
- What their biggest challenges are
- What tasks they need help with

The chat title shows: "Welcome back, [BusinessName]'s Assistant"

## New Files Created

### API Endpoint
- **`/api/analyze-user-profile`** - Uses Claude 3.5 Sonnet to analyze user descriptions and generate personalized profiles

### Updated Components
- **`OnboardingForm.tsx`** - New 6-step flow:
  - Step 0: "Tell us about your work" (free text)
  - Step 1: Business name + review AI analysis
  - Step 2: Pain points
  - Step 3: Tools
  - Step 4: Personalized task preview
  - Step 5: Confirmation

### Updated Files
- **`lib/onboarding.ts`** - New types:
  - `UserProfile` (generated by AI)
  - `WorkType`, `RevenueModel`, `WorkMode` (expanded)
  - `OnboardingData.userProfile` (optional profile data)
  - `OnboardingData.userDescription` (new field)

- **`app/dashboard/page.tsx`** - Dynamic task buttons:
  - Buttons generated from `profile.userProfile.suggestedTasks`
  - Profile shows personalized info
  - Dashboard text is fully context-aware

- **`app/chat/page.tsx`** - Profile-aware:
  - Loads user profile on mount
  - Uses profile context in response generation
  - Chat title shows business name
  - Opening message from profile context
